# 🛡️ XShield — Полная инструкция по установке и использованию

## 📦 Требования

### VPS-сервер (зарубежный)

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| **ОС** | Debian 11 / Ubuntu 22.04 | Ubuntu 24.04 |
| **CPU** | 1 vCPU | 2 vCPU |
| **RAM** | 1 GB | 2–4 GB |
| **Диск** | 10 GB SSD | 20 GB SSD |
| **Сеть** | 500 Мбит/с | 1 Гбит/с |
| **IP** | 1 IPv4 | 1 IPv4 (чистый, не заблокированный) |

**Рекомендуемые хостинги:**
- 🇳🇱 Нидерланды: VDSina, TimeWeb Cloud, Aeza
- 🇩🇪 Германия: Hetzner, Contabo
- 🇫🇮 Финляндия: UpCloud
- 🇸🇬 Сингапур: Oracle Cloud (Free Tier!)

**Стоимость:** ~400–1000 ₽/мес.

### Перед установкой

1. **Telegram Bot Token** — создайте бота через [@BotFather](https://t.me/BotFather)
2. **Ваш Telegram ID** — узнайте через [@userinfobot](https://t.me/userinfobot)
3. **DuckDNS** — зарегистрируйте бесплатный субдомен на [duckdns.org](https://www.duckdns.org)
4. **SSH-доступ** к VPS с правами root

## 🔧 Подготовка

### 1. Создание бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Введите имя бота (например: `XShield VPN`)
4. Введите username бота (например: `my_xshield_bot`)
5. **Сохраните токен** (формат: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 2. Узнайте свой Telegram ID

1. Откройте [@userinfobot](https://t.me/userinfobot)
2. Отправьте `/start`
3. **Сохраните ваш ID** (число, например: `123456789`)

### 3. Настройка DuckDNS (для HTTPS)

1. Перейдите на [duckdns.org](https://www.duckdns.org)
2. Войдите через GitHub/Google/Twitter
3. Создайте субдомен (например: `myshield`)
4. **Сохраните Token** (на главной странице сверху)
5. IP будет обновлён автоматически при установке

### 4. Подключение к VPS

```bash
ssh root@ВАШ_IP_VPS
```

Или через PuTTY (Windows) / Terminal (macOS/Linux).

## 🚀 Установка на VPS

### Быстрая установка (одна команда)

```bash
# Установите git если его нет
apt update && apt install -y git

# Клонируйте проект
git clone https://github.com/ВАШ_USERNAME/xshield.git /opt/xshield

# Запустите установку
cd /opt/xshield
chmod +x install.sh
bash install.sh
```

### Что установочный скрипт делает автоматически

| Шаг | Действие | Время |
|:---:|----------|:-----:|
| 1 | Обновление системы и установка зависимостей | ~3 мин |
| 2 | Харденинг сервера (BBR, UFW, fail2ban) | ~1 мин |
| 3 | Установка Xray-core | ~1 мин |
| 4 | Генерация ключей Reality и конфигурация | ~30 сек |
| 5 | Установка Python venv и зависимостей | ~2 мин |
| 6 | Установка Caddy (HTTPS) | ~1 мин |
| 6.5 | Сборка Mini App (React) | ~1 мин |
| 7 | Создание systemd-сервисов | ~30 сек |
| 8 | Настройка cron-задач | ~30 сек |

### Во время установки скрипт спросит

| Запрос | Что ввести | Пример |
|--------|-----------|---------|
| **Telegram Bot Token** | Токен от @BotFather | `1234567890:ABC...` |
| **Admin Telegram ID** | Ваш числовой ID | `123456789` |
| **PIN code** | 4-значный PIN для критических операций | `4829` |
| **DuckDNS subdomain** | Ваш субдомен | `myshield` |
| **DuckDNS token** | Токен с duckdns.org | `a1b2c3d4-...` |
| **MTProto Proxy?** | y/n (опционально) | `n` |
| **AdGuard Home?** | y/n (опционально) | `n` |
| **WARP?** | y/n (опционально) | `n` |

## 📱 Подключение клиентов

### 🖥️ Windows (v2rayN)

#### Установка

1. Скачайте **v2rayN** с [GitHub](https://github.com/2dust/v2rayN/releases)
   - Файл: `v2rayN-With-Core.zip`
2. Распакуйте в удобную папку (например, `C:\v2rayN`)
3. Запустите `v2rayN.exe`
4. Если требуется — установите .NET Runtime

#### Подключение через ссылку

1. Скопируйте VLESS-ссылку из бота
2. В v2rayN: **Серверы → Добавить сервер из буфера**
3. Сервер появится в списке
4. Нажмите **▶ Подключить** (или правой кнопкой → Подключить)

#### Подключение через подписку (рекомендуется)

1. Скопируйте URL подписки из бота
2. В v2rayN: **Подписка → Настройка подписки → Добавить**
3. Вставьте URL → ОК
4. **Подписка → Обновить подписку**
5. Выберите сервер → Подключить

> **💡 Преимущество подписки:** конфигурация обновляется автоматически каждые 6 часов.

#### Настройка маршрутизации в v2rayN

По умолчанию v2rayN пропускает **весь** трафик через VPN. Для раздельной маршрутизации:

1. **Настройки → Routing Settings**
2. Выберите правило: **«Обход адресов РФ»**
3. Или вручную добавьте домены для проксирования

### 📱 Android (v2rayNG)

#### Установка

1. Скачайте **v2rayNG** из:
   - [Google Play](https://play.google.com/store/apps/details?id=com.v2ray.ang)
   - [GitHub APK](https://github.com/2dust/v2rayNG/releases)

#### Подключение через QR-код

1. В боте нажмите: 👥 → выберите пользователя → 📱 QR
2. В v2rayNG: **+ → Сканировать QR-код**
3. Наведите камеру на QR-код
4. Нажмите ▶ для подключения
5. Разрешите VPN-подключение

#### Подключение через подписку

1. Скопируйте URL подписки из бота
2. В v2rayNG: **☰ → Группа подписки → +**
3. Вставьте URL → ОК
4. **Обновить подписку**
5. Нажмите ▶

### 🍎 iOS (FoXray)

#### Установка

1. Скачайте **FoXray** из [App Store](https://apps.apple.com/app/foxray/id6448898396)
   - Может потребоваться Apple ID другого региона (если FoXray недоступен в РФ)

#### Подключение

1. Скопируйте VLESS-ссылку из бота
2. В FoXray: **+ → Импорт из буфера**
3. Или отсканируйте QR-код
4. Включите VPN

#### Подключение через подписку

1. В FoXray: **Настройки → Подписки → +**
2. Вставьте URL подписки
3. Нажмите **Обновить**

### 📡 Роутер (OpenWRT)

#### OpenWRT + Passwall2

1. Установите пакет `passwall2` через LuCI
2. Перейдите: **LuCI → Services → Passwall2**
3. Добавьте новый сервер:
   - Тип: **VLESS**
   - Адрес: ваш IP сервера
   - Порт: `443`
   - UUID: (из бота)
   - Transport: `XHTTP`
   - TLS: Reality
   - SNI: `www.microsoft.com`
   - Public Key: (из бота)
   - Short ID: (из бота)
4. Включите режим **Global / GFW List**

#### Keenetic + XKEEN

1. Установите [XKEEN](https://github.com/Jenya-developer/xkeen) через SSH
2. Добавьте конфигурацию через веб-интерфейс роутера

> **💡 Совет:** Для роутера лучше использовать подписочную ссылку — настройки обновляются автоматически.

## 🔌 Дополнительные компоненты

### 💬 MTProto Proxy

**Что это:** Прокси для Telegram. Если Telegram будет заблокирован — через этот прокси бот останется доступен.

**Установка:**
```bash
bash /opt/xshield/server/mtproto/setup.sh
```

**Использование:**
1. Бот выдаст ссылку формата: `tg://proxy?server=IP&port=443&secret=...`
2. Отправьте ссылку себе/друзьям
3. При нажатии — Telegram предложит добавить прокси

### 🛡️ AdGuard Home

**Что это:** Блокировщик рекламы и трекеров на уровне DNS. Все устройства, подключённые через XShield, получат блокировку рекламы **без установки AdBlock**.

**Установка:**
```bash
bash /opt/xshield/server/adguard/setup.sh
```

**Что будет блокироваться:**
- Рекламные баннеры и видеореклама
- Google Analytics, Facebook Pixel и другие трекеры
- Фишинговые и вредоносные домены

**Панель управления:** `http://127.0.0.1:8053` (admin / xshield)

> **⚠️ Важно:** AdGuard занимает порт 53 (DNS). Скрипт автоматически отключит `systemd-resolved`.

### ☁️ Cloudflare WARP

**Что это:** Double-hop VPN. Ваш трафик идёт: Устройство → VPS → Cloudflare → Интернет. Сайты видят IP Cloudflare, а не вашего VPS.

**Зачем нужно:**
- Защита IP вашего VPS от трекинга
- Обход блокировок стриминговых сервисов (Netflix, Disney+)
- «Чистый» IP Cloudflare — нет в чёрных списках

**Установка:**
```bash
bash /opt/xshield/server/warp/setup.sh
```

**Управление через бота:**
- ⚙️ Настройки → появится кнопка WARP On/Off
- Или через Mini App → Settings → переключатель WARP

## 📡 Подписка и автообновление

### Что такое подписка

Подписка — это URL, который клиент (v2rayN/v2rayNG) периодически проверяет и автоматически обновляет конфигурацию. Вам не нужно вручную менять настройки при обновлении сервера.

### URL подписки

Формат: `https://<ваш-домен>.duckdns.org:8443/sub/ТОКЕН`

Каждый пользователь имеет уникальный токен. Получить его можно:
- При создании пользователя (бот отправит автоматически)
- В меню: 👥 → выбрать пользователя → 📡 Подписка

### Что обновляется автоматически

- ✅ Параметры сервера (IP, ключи, SNI)
- ✅ Статистика трафика (клиент видит сколько осталось)
- ✅ Срок действия аккаунта
- 🔄 Обновление каждые 6 часов

### Правила маршрутизации

Клиент может дополнительно загрузить правила маршрутизации:

```
https://<ваш-домен>.duckdns.org:8443/sub/ТОКЕН/routing
```

Это JSON с правилами, какие сайты идут через PROXY, а какие DIRECT.

## 🔥 Устранение неполадок

### Бот не отвечает

```bash
# Проверьте статус
systemctl status xshield-bot

# Посмотрите ошибки
journalctl -u xshield-bot --no-pager -n 50

# Перезапустите
systemctl restart xshield-bot
```

**Частые причины:**
- Неверный `BOT_TOKEN` в `.env`
- Нет интернета на VPS
- Python venv сломался → пересоздайте: `python3 -m venv /opt/xshield/venv`

### VPN не подключается

1. **Проверьте Xray:**
   ```bash
   systemctl status xray
   journalctl -u xray --no-pager -n 30
   ```

2. **Проверьте порт 443:**
   ```bash
   ss -tlnp | grep 443
   ```

3. **Проверьте UFW:**
   ```bash
   ufw status
   # Должно быть: 443/tcp ALLOW
   ```

4. **Убедитесь, что IP не заблокирован:**
   Через бота: 📊 → 🧪 Проверьте доступность вашего VPS

### Подписка не обновляется

1. **Проверьте subscription-сервер:**
   ```bash
   systemctl status xshield-sub
   ```

2. **Проверьте Caddy (HTTPS):**
   ```bash
   systemctl status caddy
   journalctl -u caddy --no-pager -n 20
   ```

3. **Проверьте доступность:**
   ```bash
   curl -k https://localhost:8443/health
   # Ответ: {"status": "ok"}
   ```

### Mini App не открывается

1. Убедитесь, что DuckDNS настроен и указывает на IP VPS
2. Проверьте, что Caddy получил TLS-сертификат
3. Проверьте, что webapp собран:
   ```bash
   ls /opt/xshield/server/webapp/dist/
   ```
   Если нет — пересоберите:
   ```bash
   cd /opt/xshield/server/webapp
   npm install && npm run build
   ```

### IP заблокирован ТСПУ

**Симптомы:** VPN перестал работать, но VPS доступен по SSH (через другой IP/прокси).

**Решения:**
1. **Включите WARP** — трафик пойдёт через IP Cloudflare
2. **Смените SNI** — через бота: ⚙️ → 📝 Изменить SNI
3. **Смените VPS/IP** — переустановите XShield на новом сервере, восстановите из бэкапа

## 🔐 Безопасность

### Что защищено автоматически

| Мера | Описание |
|------|----------|
| **UFW Firewall** | Открыты только порты 22, 443, 8443 |
| **fail2ban** | Защита от брутфорса SSH |
| **TCP BBR** | Оптимизация скорости сети |
| **Reality** | Протокол неотличим от HTTPS для DPI |
| **XHTTP** | Новейший транспорт, устойчивый к обнаружению |
| **PIN-код** | Критические операции требуют подтверждения |

### Рекомендации по безопасности

1. **Используйте сложный PIN** — не `0000` или `1234`
2. **Ограничьте Admin IDs** — только ваш Telegram ID
3. **Регулярно обновляйте** — Xray-core и гео-базы
4. **Делайте бэкапы** — автоматические ежедневно + ручные перед изменениями
5. **Не делитесь VLESS-ссылками** в открытых чатах
6. **Используйте подписку** вместо прямых ссылок — при компрометации можно пересоздать токен
7. **Мониторьте логи** — если увидите подозрительную активность

### Файл .env (конфиденциальные данные)

Файл `/opt/xshield/.env` содержит все секретные данные. Он защищён правами `600` (только root).

```bash
# Проверьте права
ls -la /opt/xshield/.env
# -rw------- 1 root root ... .env
```

## ❓ FAQ

**Q: Это бесплатно?**
A: Всё ПО бесплатное и open-source. Единственная затрата — VPS (~400-1000 ₽/мес).

**Q: Сколько пользователей можно добавить?**
A: Практически без ограничений. На VPS с 1 vCPU / 2 GB RAM комфортно работают 10-20 пользователей.

**Q: Нужен ли отдельный домен?**
A: Нет! DuckDNS предоставляет бесплатный субдомен. Но вы можете использовать свой домен.

**Q: Что будет, если VPS заблокируют?**
A: IP сервера может быть заблокирован ТСПУ. Решение: включить WARP, сменить SNI, или переехать на новый VPS (восстановление из бэкапа).

**Q: Можно ли использовать на роутере?**
A: Да! Поддерживаются OpenWRT (Passwall2) и Keenetic (XKEEN). VPN для всей домашней сети.

**Q: VPN замедляет интернет?**
A: Минимально. Благодаря раздельной маршрутизации через VPN идут только заблокированные сайты. Остальной трафик — напрямую.
